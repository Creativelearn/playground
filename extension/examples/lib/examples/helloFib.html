The first example should explain the most basic techniques used in jsreport.

Assembling reports in jsreport is based on javascript templating engines. You define a html template with special tags like this one and jsreport will use templating engines rendering and creates report output for you. Output can be html, pdf and others will come. This example is rendering simple html, but you will learn about other output formats in the next examples.

In the integrated template designer is the template on the left side. You can find a preview of the final report on the right side. Using "Run" button from the toolbar will invoke post to the jsreport server and displays the output in the preview pane. Go ahead and try it!

Every template can contains javascript helpers. This comes from basic concepts of javascript templating engines. The content tab contains the report design and the helpers can contain any javascript function that can do some computations required in the report. Go ahead and switch "Content" and "Helpers" tabs to see whats there.
